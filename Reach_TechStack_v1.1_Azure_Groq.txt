2. AI Provider — Groq Cloud
Groq replaces Anthropic Claude in this stack. The free tier provides ~500 requests/day at near-zero latency using llama-3.3-70b-versatile — ideal for the two AI tasks in REACH: segment intent parsing and message drafting.

Attribute
Detail
Provider
Groq Cloud — https://console.groq.com
Model
llama-3.3-70b-versatile (recommended) or mixtral-8x7b-32768 as fallback
Free Tier
~14,400 tokens/min, ~500 req/day — sufficient for demo and evaluation use cases
Latency
< 500ms for a typical segment filter JSON response — Groq's hardware inference delivers sub-second on most prompts
Use Case 1
Segment intent parsing: natural language → structured JSON filter ({ recency_days: 90, min_orders: 2, lifecycle: 'at_risk' })
Use Case 2
Message drafting: campaign goal + audience context → personalized copy with {{first_name}}, {{last_product}}, {{days_since_purchase}} tokens
API Key
Set as GROQ_API_KEY environment variable in Azure Static Web Apps application settings and App Service config
Swap path
AI calls are abstracted behind a /lib/ai.ts module. Swapping to GPT-4o or Gemini requires only changing the provider constructor — no other code changes.

2.1  Groq API Call Pattern
// lib/ai.ts — Groq provider abstraction
import Groq from 'groq-sdk';
const groq = new Groq({ apiKey: process.env.GROQ_API_KEY });

export async function parseSegmentIntent(nl: string) {
  const response = await groq.chat.completions.create({
    model: 'llama-3.3-70b-versatile',
    messages: [{ role: 'user', content: buildSegmentPrompt(nl) }],
    response_format: { type: 'json_object' } // structured output
  });
  return JSON.parse(response.choices[0].message.content);
}
